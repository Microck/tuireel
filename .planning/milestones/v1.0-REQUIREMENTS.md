# Requirements: Tuireel

**Defined:** 2025-07-26
**Core Value:** TUI tool authors can produce polished demo videos from a declarative script, without manual screen recording, editing, or post-production.

## v1 Requirements

Requirements for initial release. Each maps to roadmap phases.

### Recording Core

- [x] **REC-01**: User can define a terminal command to launch in a virtual PTY
- [x] **REC-02**: User can capture pixel-perfect terminal frames as images via tuistory
- [x] **REC-03**: User can encode frames to MP4 video
- [x] **REC-04**: User can encode frames to WebM video
- [x] **REC-05**: User can encode frames to animated GIF with optimized palette
- [x] **REC-06**: User can configure frame rate (default 30fps)
- [x] **REC-07**: User can configure terminal viewport (cols/rows)
- [x] **REC-08**: ffmpeg is auto-downloaded to ~/.tuireel on first use

### Step Types

- [x] **STEP-01**: User can type text character-by-character with configurable speed
- [x] **STEP-02**: User can press keys/chords (Enter, Ctrl+C, arrow keys, etc.)
- [x] **STEP-03**: User can wait for text/regex pattern to appear in terminal
- [x] **STEP-04**: User can pause for a fixed duration
- [x] **STEP-05**: User can scroll within the TUI (mouse scroll events)
- [x] **STEP-06**: User can click on a text pattern in the terminal
- [x] **STEP-07**: User can capture a PNG screenshot at any point
- [x] **STEP-08**: User can resize terminal dimensions mid-recording
- [x] **STEP-09**: User can set environment variables mid-script

### Overlay System

- [x] **OVR-01**: User can enable animated cursor overlay with human-like Bezier movement
- [x] **OVR-02**: User can enable keystroke HUD showing pressed keys as visual overlay
- [x] **OVR-03**: User can re-composite overlays without re-recording (composite command)
- [x] **OVR-04**: InteractionTimeline is saved to disk enabling re-compositing

### Terminal Theming

- [x] **THM-01**: User can select from built-in themes (Dracula, Catppuccin, One Dark, Monokai, Solarized, Tokyo Night, Nord, Gruvbox)
- [x] **THM-02**: User can define custom themes (colors, font, padding, background)

### Sound

- [x] **SND-01**: User can enable click/key sound effects synced to actions
- [x] **SND-02**: User can mix in a custom audio track

### Config & CLI

- [x] **CLI-01**: User can define videos in JSONC config with comments
- [x] **CLI-02**: Config files have JSON Schema for IDE autocompletion
- [x] **CLI-03**: User can run `tuireel record` to produce video
- [x] **CLI-04**: User can run `tuireel preview` to execute steps without recording
- [x] **CLI-05**: User can run `tuireel validate` to check config for errors
- [x] **CLI-06**: User can run `tuireel init` to scaffold a starter config
- [x] **CLI-07**: User can run `tuireel composite` to re-render overlays from saved timeline
- [x] **CLI-08**: Config re-records automatically in watch mode on file change
- [x] **CLI-09**: One config file can define multiple videos
- [x] **CLI-10**: Step sequences can be shared/included across videos

### Infrastructure

- [x] **INF-01**: Package installable via npm (npx tuireel) and Bun (bunx tuireel)
- [x] **INF-02**: Monorepo with @tuireel/core (engine) and tuireel (CLI)

## Out of Scope

Explicitly excluded. Documented to prevent scope creep.

| Feature | Reason |
|---------|--------|
| Live/interactive recording | Defeats scripted repeatability. Use asciinema. |
| asciicast/.cast output | Different paradigm. We produce video, not text recordings. |
| Web player | We produce video files that play anywhere. |
| Live streaming | Out of scope. asciinema handles this. |
| Browser recording | webreel does this. We do terminals. |
| Windows support | tuistory is Unix-only (PTY). macOS + Linux only. |
| Built-in video editor | Produce files; use external tools to edit. |
| Custom scripting language | VHS created tape format. JSONC is simpler and toolable. |
