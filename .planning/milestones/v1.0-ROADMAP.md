# Milestone v1.0: MVP

**Status:** SHIPPED 2026-03-03
**Phases:** 1-6
**Total Plans:** 23

## Overview

v1.0 delivered Tuireel's complete first release scope: scripted TUI recording from declarative config, multi-format video output (MP4/WebM/GIF), optional overlays and sound, and workflow tooling for preview, watch mode, and multi-video authoring.

## Milestone Summary

**Decimal phases:** None

**Key decisions:**
- Keep a two-pass architecture: record clean timeline artifacts first, composite overlays afterward.
- Use tuistory as the terminal automation/rendering engine instead of re-implementing PTY and screenshot internals.
- Maintain JSONC + JSON Schema as the authoring contract for validation and IDE autocomplete.
- Keep sound opt-in so existing scripts remain silent by default.
- Separate authoring config loading from runtime normalized config loading for multi-video support.
- Apply runtime `set-env` via shell export emission and idle waits to make behavior observable and deterministic.

**Issues resolved during milestone:**
- Fixed Turborepo workspace resolution via required root `packageManager` metadata.
- Closed wait-regex contract gap between config schema and executor.
- Corrected viewport/frame behavior to avoid terminal content trim side effects.
- Raised cursor jitter envelope for long movements to satisfy overlay realism criteria.
- Added runtime propagation and regression coverage for `set-env` behavior.
- Aligned CLI packaging for real `npx`/`bunx` execution paths.

**Issues deferred:**
- tuistory API instability risk remains (tracked for future hardening).
- Frame timing consistency under all terminal workloads needs broader soak coverage.
- ffmpeg pipeline fragility risk remains for edge-case environments.
- GIF size/quality optimization can be further tuned beyond baseline two-pass output.

**Technical debt incurred:**
- Risk mitigations are documented but not fully operationalized as runtime SLO checks.
- End-to-end stress tests for long watch sessions remain limited.
- Publishing/release automation is not yet codified.
