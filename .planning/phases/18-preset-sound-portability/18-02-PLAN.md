  <name>Task 2: Run non-repo record smoke (npx + bunx) for presets polished + demo and assert audio stream exists</name>
  <files>scripts/publish-smoke.ts</files>
  <action>
Add an E2E record check in the existing temp install directory (`npxDir`).

Constraints:

- The smoke must run from a temp directory (non-repo CWD) and use installed tarballs.
- Config must be minimal but must generate at least one key event so SFX mixing runs.

Implementation details:

- In `npxDir` (and in `bunDir` when bun is available), write 2 config files:
  - `sound-smoke-polished.jsonc` with `{ "preset": "polished", "output": "sound-smoke-polished.mp4", "steps": [...] }`
  - `sound-smoke-demo.jsonc` with `{ "preset": "demo", "output": "sound-smoke-demo.mp4", "steps": [...] }`
- Steps: launch `bash`, type `echo sound`, press `enter`, wait for `sound`, short pause.
- Run (cwd = npxDir):
  - `npx tuireel record ./sound-smoke-polished.jsonc --format mp4`
  - `npx tuireel record ./sound-smoke-demo.jsonc --format mp4`
- Assert audio stream exists by running the downloaded ffmpeg binary for both outputs:
  - `~/.tuireel/bin/ffmpeg -v error -i sound-smoke-polished.mp4 -map 0:a:0 -f null -`
  - `~/.tuireel/bin/ffmpeg -v error -i sound-smoke-demo.mp4 -map 0:a:0 -f null -`
  - Treat non-zero exit as failure (no audio stream).

- When bun is available, extend the existing bun temp install (`bunDir`) to also record via bunx (cwd = bunDir):
  - `bunx --bun --no-install tuireel record ./sound-smoke-polished.jsonc --format mp4`
  - `bunx --bun --no-install tuireel record ./sound-smoke-demo.jsonc --format mp4`
  - Reuse the same ffmpeg audio stream assertions for the bun outputs.

- When bun is not available, report a passing "skipped" check for bunx record (do not fail the smoke).

Keep existing `--help` smoke checks; this is additive.
</action>
