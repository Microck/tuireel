  <name>Task 1: Implement tarball pack + install smoke script</name>
  <files>scripts/publish-smoke.ts</files>
  <action>
Create a TypeScript smoke script that validates the *publish artifact* rather than the workspace.

Behavior:

- Create a temp directory.
- Run BOTH packs into the same destination:
  - `pnpm --filter tuireel pack --pack-destination <temp>/packs`
  - `pnpm --filter "@tuireel/core" pack --pack-destination <temp>/packs`
- Locate the produced `tuireel-*.tgz` and `tuireel-core-*.tgz`.
- For EACH tarball, read `package/package.json` from the gzipped tarball (shell out to `tar -xOzf ... package/package.json`) and assert it does NOT contain `workspace:`.
- Run `npx -y <tuireel-tarball> --help` with `cwd` set to a fresh temp directory and assert exit code 0.

Bun coverage (REL-04):

- If `bun` is present in PATH, run an additional clean-environment check:
  - Create a second temp directory.
  - Run `bun add <tuireel-tarball>`.
  - Execute the installed CLI and assert exit code 0:
    - Prefer: `./node_modules/.bin/tuireel --help` and `./node_modules/.bin/tuireel --version`
  - Do not require `bunx` here; bunx is validated post-publish in the release workflow (Task 3).

Constraints:

- No new dependencies.
- Keep output actionable: if failing, print which check failed and include the relevant snippet (e.g. the dependency line or the stderr tail).
  </action>
  <verify>
  pnpm -w build --filter tuireel...
  pnpm -w exec tsx scripts/publish-smoke.ts
  </verify>
  <done>`pnpm -w exec tsx scripts/publish-smoke.ts` exits 0 locally when the package is pack/installable.</done>

<task type="auto">
  <name>Task 3: Gate the release workflow with publish smoke</name>
  <files>.github/workflows/release.yml</files>
  <action>
Wire the smoke check into `.github/workflows/release.yml` so publishing cannot proceed if installability is broken.

- Ensure Bun is available before running the smoke check (tarball bun-install coverage): add a step using `oven-sh/setup-bun@v2`.
- Add a step after `pnpm build` and before `changesets/action@v1` to run the smoke check.
- Run: `pnpm -w publish:smoke`.
- After the `changesets/action@v1` step, add a post-publish bunx validation step guarded by publish output:
  - Guard: `if: steps.changesets.outputs.published == 'true'`
  - Setup Bun (inside the guarded step block or as a guarded setup step): `oven-sh/setup-bun@v2`
  - Run post-publish validation against the _published_ version (not tarball), with retries to avoid npm propagation flakiness:
    - Determine version: `VERSION=$(node -e "console.log(require('./packages/cli/package.json').version)")`
    - Retry loop (5 attempts, backoff):
      - `npx -y tuireel@${VERSION} --help`
      - `npx -y tuireel@${VERSION} --version`
      - `bunx tuireel@${VERSION} --help`
      - `bunx tuireel@${VERSION} --version`
- Keep existing trusted-publishing setup unchanged.
  </action>
