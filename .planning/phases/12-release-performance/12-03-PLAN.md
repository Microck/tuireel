<task type="auto">
  <name>Task 2: Profile and apply one compositing optimization</name>
  <files>packages/core/src/compositor.ts</files>
  <action>
Read `packages/core/src/compositor.ts` thoroughly. Profile the compositing pipeline to identify the biggest bottleneck.

**Likely optimization candidates (investigate in order):**

1. **Skip unchanged overlay frames**: If cursor position and HUD state haven't changed between frames, reuse the previous composited frame buffer instead of re-compositing. This is the most impactful optimization for recordings with pauses.

2. **Batch Sharp operations**: If the compositor calls Sharp per-frame for overlay compositing, batch operations or reuse intermediate buffers where overlays haven't changed.

3. **Reduce decode overhead**: If the compositor re-decodes frames unnecessarily, cache decoded frame data.

**Process:**
1. Run benchmark BEFORE optimization: record the baseline numbers
2. Read compositor.ts to understand the frame processing loop
3. Identify the highest-impact optimization from the candidates above
4. Implement it
5. Run benchmark AFTER optimization: record the improved numbers
6. Add a comment in the code noting the optimization and its measured impact

**The optimization must be measurable.** If the benchmark shows < 5% improvement, the optimization is not worth the complexity — try a different one.

**Important:** Do NOT break existing behavior. The compositor must produce identical output before and after optimization. Run `pnpm test` to verify.
  </action>
  <verify>
`pnpm test` passes (no behavioral regression). `pnpm benchmark` shows measurable improvement in compositing time compared to baseline. The optimization is documented with a code comment.
  </verify>
  <done>At least one compositing optimization applied with measurable improvement documented against benchmark baseline.</done>
</task>
