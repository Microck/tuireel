**Steps reference (`docs/steps-reference.mdx`):**
Read the step schemas from `packages/core/src/config/schema.ts`. Document each step type:

- `launch`: command (string, required). Starts a terminal program.
- `type`: text (string), speed (number, optional). Types text with optional speed.
- `press`: key (string). Sends a key press (Enter, Tab, Ctrl+C, etc.).
- `wait`: pattern (string or regex), timeout (number, optional). Waits for text to appear.
- `pause`: duration (number). Pauses for N milliseconds.
- `scroll`: direction (up|down), lines (number, optional). Scrolls the terminal.
- `click`: x (number), y (number). Clicks at coordinates.
- `screenshot`: (no fields). Takes a snapshot at this point.
- `resize`: cols (number), rows (number). Resizes the terminal.
- `set-env`: env (Record<string, string>). Sets environment variables.

For each step type: JSON example, description of behavior, all fields with types.
Include a combined example showing a realistic multi-step config.

**CLI reference (`docs/cli-reference.mdx`):**
Read the CLI command files to get exact flags. Document each command:

- `tuireel init` — scaffold a new config file (preset selection in TTY mode)
- `tuireel validate <config>` — validate config against schema
- `tuireel record <config>` — record a TUI session to video
  - `--format <mp4|webm|gif>` — override output format
  - `-w, --watch` — watch config and re-record on changes
  - `--verbose` — show step-by-step progress (from Phase 9)
  - `--debug` — show full ffmpeg/tuistory output (from Phase 9)
- `tuireel preview <config>` — run script in visible terminal without recording
- `tuireel composite <config>` — re-composite with overlays without re-recording

For each command: usage syntax, all flags, description, example invocations.

**Read the actual command files to get exact flag definitions. Do NOT guess.**
