  1. Copy bundled sound assets from webreel reference (`opensrc/repos/github.com/vercel-labs/webreel/packages/@webreel/core/assets/*.mp3`) into `packages/core/assets/sounds/`. Same naming: key-1..4.mp3, click-1..4.mp3.

  2. Add `SoundEvent` type to `packages/core/src/types.ts` if not already present:
     - `{ type: "click" | "key"; timeMs: number }`

  3. Create `packages/core/src/sound.ts` following webreel's `media.ts` pattern:
     - `SfxConfig` interface: `{ click?: 1|2|3|4|string; key?: 1|2|3|4|string }`
     - `resolveSfxPath(value, prefix)` — resolves numbered preset or custom path
     - `ensureSoundAssets(sfx?)` — returns `{ clickPath, keyPath }`
     - `extractSoundEvents(timeline)` — extracts SoundEvent[] from InteractionTimeline's events
     - `buildAudioMixArgs(videoInput, events, durationSec, sfx?)` — builds ffmpeg `-i` args and `-filter_complex` string using:
       - `anullsrc` silence base track
       - Per-event: `adelay=Nms|Nms` for placement, `asetrate` for subtle pitch variation, `volume` for humanization
       - Final `amix=inputs=N+1:normalize=0` to combine
     - `finalizeMp4WithSound(ffmpegPath, tempVideo, outputPath, events, durationSec, sfx?)` — if events empty or no sfx, just rename/remux. Otherwise run ffmpeg with audio mix args, `-c:v copy -c:a aac -b:a 128k -shortest -movflags +faststart`.
     - `finalizeWebmWithSound(...)` — same pattern but `-c:a libopus`.
     - Keep `runFfmpeg` helper (spawnSync, throw on non-zero exit).

  4. Export from `packages/core/src/index.ts`: `SfxConfig`, `extractSoundEvents`, `buildAudioMixArgs`, `ensureSoundAssets`, `finalizeMp4WithSound`, `finalizeWebmWithSound`.

  Key differences from webreel: Use `extractSoundEvents` as explicit function (webreel inlines it). Sound types include "key" for all keyboard events (enter, backspace are key variants, not separate types — keep it simple like webreel).
  </action>
