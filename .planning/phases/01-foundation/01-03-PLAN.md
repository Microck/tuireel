<task type="auto">
  <name>Task 1: Implement ffmpeg downloader</name>
  <files>packages/core/src/ffmpeg/downloader.ts, packages/core/src/ffmpeg/index.ts</files>
  <action>
  - getFfmpegPath(): returns ~/.tuireel/bin/ffmpeg (use os.homedir())
  - detectPlatform(): returns {os: "darwin"|"linux", arch: "arm64"|"x64"} from process.platform/arch
  - getDownloadUrl(platform): map to static build URLs:
    - darwin-arm64/x64: https://evermeet.cx/ffmpeg/getrelease/ffmpeg/zip (or GitHub releases from BtbN/FFmpeg-Builds for cross-platform)
    - linux-x64: https://johnvansickle.com/ffmpeg/ static build or BtbN GitHub releases
    - Use BtbN/FFmpeg-Builds GitHub releases (more reliable, all platforms): `https://github.com/BtbN/FFmpeg-Builds/releases/download/latest/ffmpeg-master-latest-{os}{arch}-gpl.tar.xz`
  - ensureFfmpeg(): check if binary exists + is executable (fs.access with X_OK). If yes, return path. If no: mkdir -p ~/.tuireel/bin, download archive, extract ffmpeg binary, chmod +x, verify with execFile("ffmpeg", ["-version"]), return path.
  - Use lockfile (~/.tuireel/bin/.ffmpeg.lock) to prevent concurrent download race condition. Simple approach: check for .lock file, wait if exists, create before download, remove after.
  - Log progress to stderr (downloading..., extracting..., done)
  - ffmpeg/index.ts: barrel export ensureFfmpeg, getFfmpegPath
  - Update packages/core/src/index.ts to re-export ffmpeg module
  </action>
  <verify>
  Run ensureFfmpeg() — binary appears at ~/.tuireel/bin/ffmpeg. Run again — skips download. Run `~/.tuireel/bin/ffmpeg -version` — prints version.
  </verify>
  <done>ffmpeg auto-downloads on first call, caches for subsequent calls, binary is executable.</done>
</task>
