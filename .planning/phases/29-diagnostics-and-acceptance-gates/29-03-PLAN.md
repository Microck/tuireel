  <files>packages/core/test/acceptance/pacing.acceptance.ts</files>
  <action>
    Create `packages/core/test/acceptance/pacing.acceptance.ts`:

    1. Setup: In `beforeAll`, create a temp directory, copy the smoke fixture into it (or create a dedicated acceptance fixture JSONC inline with known type/pause steps), run `record()` from `@tuireel/core` with a config that has known pause durations and typed text.

    2. The acceptance fixture should include:
       - A `launch` step
       - A `pause` step with a known duration (e.g. 500ms)
       - A `type` step with known text (e.g. "echo hello world" — 16 chars)
       - A `press` step (Enter)
       - Another `pause` step with a different known duration (e.g. 300ms)
       This gives clear pacing signals to assert on.

    3. Tests:
       - "timeline records expected frame count range": Load timeline, assert `frameCount > 0` and `frameCount < 1000` (sanity bound for a short demo).
       - "typed characters produce multiple keyframes": Load timeline, look at `frames` array. Typing 16 chars should produce multiple keyframes (not just 1). Assert `frames.length >= 3` (at minimum: start, some typing frames, end).
       - "pause durations are reflected in timeline timing": Look at frame timing gaps in the timeline. The 500ms pause should correspond to ~5 frames at 10fps. Assert there exists a gap between consecutive keyframes that spans roughly 400-600ms (tolerance).
       - "total recording duration is in expected range": The fixture has ~1300ms of explicit pauses plus typing time. Assert total duration (frameCount / fps * 1000) is between 1000ms and 5000ms.

    4. Cleanup: `afterAll` removes the temp directory.

    5. Important: Use `describe("acceptance: pacing", ...)` naming. All assertions use ranges, not exact values, per research guidance on timing variance. Set vitest timeout to 30000ms for the describe block since real recording takes time.
