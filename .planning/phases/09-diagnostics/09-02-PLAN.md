  <action>
Grep all `throw new Error` and `catch` blocks in `packages/core/src/`. For each error path, ensure the message follows the pattern: `"What went wrong. Try: actionable suggestion."`

**Error categories and guidance patterns:**

1. **Config errors** (loader.ts, schema validation):
   - Missing config: `"Config file not found: {path}. Try: run 'tuireel init' to create one, or check the file path."`
   - Invalid JSON: `"Failed to parse {path}: {parseError}. Try: validate JSON syntax in your editor or run 'tuireel validate {path}'."`
   - Schema validation: `"Config validation failed: {zodError}. Try: check the field against the JSON schema at {schemaUrl}."`

2. **Session errors** (session.ts):
   - Type failed: `"Failed to type text in terminal session. Try: ensure the launched program is ready for input; add a 'wait' step before typing."`
   - Press failed: `"Failed to send key '{key}' to terminal session. Try: verify the key name is correct (e.g., 'Enter', 'Tab', 'Ctrl+C')."`
   - Wait timeout: `"Timed out waiting for '{pattern}' after {timeout}ms. Try: increase the timeout, check the expected text, or add '--debug' to see terminal output."`

3. **FFmpeg errors** (compositor.ts, encoder.ts, downloader.ts):
   - Download failed: `"Failed to download ffmpeg from {url} (status {code}). Try: check your internet connection, or install ffmpeg manually and add it to PATH."`
   - Encode failed: include the ffmpeg command that was run and stderr, plus `"Try: run with '--debug' to see full ffmpeg output."`
   - Zero frames: `"Compositor decoded zero frames from {path}. Try: verify the raw recording exists and is not empty; re-record with 'tuireel record'."`

4. **Sound errors** (sound.ts):
   - Missing audio file: `"Audio file not found: {path}. Try: check the 'sound.track' path in your config is relative to the config file."`

5. **Theme errors** (themes/resolve.ts):
   - Invalid theme: `"Theme '{name}' not found. Try: use one of the built-in themes: {list}, or provide a custom theme object."`

6. **Preview errors** (preview.ts):
   - Unsupported step: `"Unsupported step type: {type}. Try: check available step types in the docs or run 'tuireel validate'."`

Preserve existing `{ cause: error }` chaining everywhere. Do NOT change error class types (keep as Error, not custom classes). Keep messages concise — one sentence for what happened, one for what to try.
