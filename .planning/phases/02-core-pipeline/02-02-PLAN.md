<task type="auto">
  <name>Task 1: Frame capturer and ffmpeg encoder</name>
  <files>
    packages/@tuireel/core/src/capture/frame-capturer.ts
    packages/@tuireel/core/src/encoding/encoder.ts
  </files>
  <action>
    Create `FfmpegEncoder` class:
    - Constructor takes `{ffmpegPath, fps, outputPath, crf?}`.
    - Spawns ffmpeg: `["-y", "-f", "image2pipe", "-framerate", fps, "-c:v", "mjpeg", "-i", "pipe:0", "-c:v", "libx264", "-preset", "fast", "-crf", crf||"23", "-pix_fmt", "yuv420p", "-movflags", "+faststart", "-r", fps, outputPath]`.
    - Use `child_process.spawn` (NOT execa — keep it simple).
    - `writeFrame(jpegBuffer)`: Write to stdin. Handle backpressure — if `write()` returns false, await `drain` event before next write. Track drain state with a promise/resolver pattern (see webreel's recorder.ts lines 120-126).
    - `finalize()`: Close stdin, await ffmpeg process exit. Reject if exit code !== 0.
    - Collect stderr for error reporting.
    - Register cleanup: on process error/exit, clean up resources.

    Create `FrameCapturer` class:
    - Constructor takes `{session: TuireelSession, encoder: FfmpegEncoder, fps: number}`.
    - `start()`: Begin interval loop at `1000/fps` ms. Each tick: call `session.screenshot()` → `encoder.writeFrame(buffer)`.
    - Handle frame timing: track wall-clock time. If a capture+encode takes longer than the interval, skip the next tick (don't queue). This prevents timing drift (Pitfall 1 from research).
    - `stop()`: Clear interval, return. Do NOT finalize encoder here — let orchestrator do that.
    - Emit frame count for progress tracking.
  </action>
  <verify>
    - `tsc --noEmit` passes
    - ffmpeg args match webreel's proven pattern (image2pipe + mjpeg input + libx264 output)
  </verify>
  <done>
    FfmpegEncoder spawns ffmpeg with image2pipe, handles backpressure. FrameCapturer runs a screenshot loop at target FPS with drift prevention.
  </done>
</task>

    Create `record(config: ValidatedConfig)` orchestrator in `recorder.ts`:
    1. Resolve ffmpeg path via `ensureFfmpeg()` (from Phase 1's ffmpeg module).
    2. Create session: `createSession({command: config.launch, cols: config.viewport.cols, rows: config.viewport.rows})`.
    3. Create encoder: `new FfmpegEncoder({ffmpegPath, fps: config.fps || 30, outputPath: config.output})`.
    4. Create capturer: `new FrameCapturer({session, encoder, fps: config.fps || 30})`.
    5. Start capture loop: `capturer.start()`.
    6. Execute steps: `await executeSteps(session, config.steps)`.
    7. After steps complete, wait one more idle cycle for final state.
    8. Stop capture: `capturer.stop()`.
    9. Finalize encoder: `await encoder.finalize()`.
    10. Close session: `session.close()`.
    - Wrap in try/finally: ALWAYS close session and kill ffmpeg on error (Pitfall 4).
    - Register SIGINT/SIGTERM handlers for cleanup.

    Export `record` from `packages/@tuireel/core/src/index.ts`.

    - Call `record(config)`.
    - Print output path on success.
    - Handle errors with user-friendly messages.
