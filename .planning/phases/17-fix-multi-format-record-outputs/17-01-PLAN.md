  <action>
Create a single helper in core that normalizes an output path to match a selected format.

Contract (implement exactly):

- When a `format` is provided, the returned path MUST end with `.<format>`.
- If the input path has no extension: append `.<format>`.
- Else: replace ONLY the last extension with `.<format>` (e.g. `name.tar.gz` + `webm` -> `name.tar.webm`).

Implementation details:

- Put the helper at `packages/core/src/utils/output-path.ts` and name it `resolveOutputPath(outputPath: string, format: OutputFormat): string`.
- Use `node:path` `extname()`; do not hand-roll path parsing.
- Export it from `packages/core/src/index.ts` so CLI can import it from `@tuireel/core`.

Tests (no mocks):

- Add `packages/core/src/__tests__/output-path.test.ts` using vitest.
- Table-driven cases at minimum:
  - `smoke-output.mp4` + `webm` -> `smoke-output.webm`
  - `smoke-output` + `gif` -> `smoke-output.gif`
  - `dir/name.webm` + `mp4` -> `dir/name.mp4`
  - `name.tar.gz` + `webm` -> `name.tar.webm`
  - Idempotence: `smoke-output.webm` + `webm` -> `smoke-output.webm`
    </action>
    <verify>
    pnpm -w test
    </verify>
    <done>Core exports `resolveOutputPath()` and unit tests pass, enforcing the normalization rules.</done>
