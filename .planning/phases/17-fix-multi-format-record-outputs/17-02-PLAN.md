<task type="auto">
  <name>Task 2: Make validation explicitly cover all three distinct outputs</name>
  <files>.github/workflows/video-smoke.yml</files>
  <action>
Make the CI-04 validation explicitly assert codec + resolution + duration for MP4, WebM, and GIF.

- Keep the existing per-format `ffprobe ... -show_format -show_streams smoke-output.<ext> > <fmt>-probe.json` steps.
- For each format's node assertion block, ensure all of the following are asserted:
  - Codec expectations (unchanged):
    - MP4: `h264`
    - WebM: `vp8` or `vp9`
    - GIF: `gif`
  - Resolution: `parseInt(v.width) > 0` AND `parseInt(v.height) > 0`
  - Duration: use a robust duration check that does NOT assume `p.format.duration` is present/valid.
    - Implement a helper in the node assertion block (inline JS is fine) that computes `durationSeconds` with this fallback chain:
      1. `p.format.duration` if finite and > 0
      2. video stream `v.duration` if finite and > 0
      3. computed duration: `parseInt(v.nb_frames) / fps` where `fps` is parsed from `v.avg_frame_rate` or `v.r_frame_rate` (handle `"30000/1001"` style values)
    - Treat missing values, `"N/A"`, empty strings, and `0` as invalid and continue to the next fallback.
    - Assert `durationSeconds > 0` (or a small epsilon like `> 0.01` if needed for very short clips).

Note: WebM/GIF duration fields can be missing/`N/A`/`0` depending on container and ffprobe version; this is why the fallback chain is required.
</action>
