<task type="auto">
  <name>Task 2: Keystroke HUD renderer</name>
  <files>packages/core/src/overlay/hud-renderer.ts</files>
  <action>
    Create `renderHud(opts)` returning `Promise<OverlayImage>`:

    Accepts:
    - `labels: string[]` — key names to display (e.g. ["Ctrl", "C"])
    - `opacity: number` — 0-1, for fade animation
    - `frameWidth: number` — video width, for centering
    - `frameHeight: number` — video height, for positioning
    - `config?: HudConfig` — styling overrides (merge with DEFAULT_HUD_CONFIG)

    Implementation:
    - Build SVG string with horizontal flex layout of key badges
    - Each badge: rounded rect background + centered text label
    - Multiple keys separated by "+" spacer
    - Apply opacity to the overall group
    - Position: centered horizontally, config.padding px from bottom (or top)
    - Convert SVG → PNG via Sharp
    - Return OverlayImage with x/y positioning for the full frame

    Badge sizing: each key label width = max(fontSize * 2, textLength * fontSize * 0.7), height = fontSize * 2.2
    Total SVG width = sum of badge widths + gaps. Height = badge height + padding.

    Keep SVG generation simple — no external font loading. Use system monospace.
  </action>
  <verify>Manual: call renderHud({labels:["Ctrl","C"], opacity:1, frameWidth:1280, frameHeight:720}), verify non-empty PNG buffer returned with correct positioning</verify>
  <done>renderHud() produces positioned PNG overlay with styled key badges, supporting opacity and configurable appearance.</done>
</task>
