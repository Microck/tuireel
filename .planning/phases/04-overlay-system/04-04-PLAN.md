<task type="auto">
  <name>Task 1: Compositor engine</name>
  <files>packages/core/src/compositor.ts</files>
  <action>
    Create `compose(rawVideoPath, timelineData, outputPath, opts?)`:

    Pipeline:
    1. Load TimelineData (already parsed JSON, passed in)
    2. Decode raw video to frame images using ffmpeg: `ffmpeg -i raw.mp4 -f image2 frames/%06d.jpg`
       - Use temp directory for extracted frames
    3. For each frame, read the corresponding FrameData from timeline:
       - If frame has cursor state (visible=true): call renderCursor(), add to overlay list
       - If frame has HUD state (non-null): call renderHud() with labels and opacity, add to overlay list
    4. Composite overlays onto frame using Sharp: `sharp(frameJpeg).composite(overlays).jpeg({quality:90}).toBuffer()`
    5. Write composited frames to output temp dir
    6. Re-encode composited frames via ffmpeg: `ffmpeg -framerate {fps} -i frames/%06d.jpg -c:v libx264 output.mp4`
       - Support format selection via outputPath extension or opts.format
       - Reuse existing encoder profiles from Phase 3 (encoder-profiles.ts)
    7. Clean up temp directories

    Opts: `{ format?: "mp4"|"webm"|"gif"; cursorConfig?: CursorConfig; hudConfig?: HudConfig }`
    Allow overriding overlay appearance at composite time (key differentiator — change settings without re-recording).

    Cache renderCursor() result (cursor image is same every frame, only position changes).
    For HUD: re-render when labels or opacity change between frames.
  </action>
  <verify>Unit: call compose() with a short test video + timeline → output file exists and is playable</verify>
  <done>compose() produces overlaid video from raw video + timeline data. Supports all three output formats.</done>
</task>
