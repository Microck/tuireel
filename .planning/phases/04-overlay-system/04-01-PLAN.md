  <behavior>
    InteractionTimeline records per-frame overlay state via a tick() mechanism:
    - tick() advances one frame, snapshots current cursor + HUD state
    - setCursorPath(points) registers a Bezier path; tick() consumes one point per frame
    - showHud(labels) / hideHud() controls keystroke HUD visibility
    - addEvent("click"|"key") records a sound event at current frame time
    - toJSON() → TimelineData (serializable), static load(json) → InteractionTimeline
    - save(path) writes JSON to disk, static fromFile(path) reads and loads

    Cases:
    - new timeline + 3 ticks → frameCount === 3, frames array has 3 entries
    - setCursorPath([p1,p2,p3]) + 3 ticks → cursor moves through p1,p2,p3
    - toJSON() → load(json) → toJSON() produces identical output (roundtrip)
    - save(path) + fromFile(path) produces identical TimelineData
    - showHud(["Ctrl","C"]) + tick → frame.hud.labels === ["Ctrl","C"]
    - hideHud() + tick → frame.hud === null
    - addEvent("key") → events array has entry with correct timeMs
  </behavior>
  <implementation>
    Model after webreel's InteractionTimeline but adapted for terminal context:
    - No zoom/viewport pixel dimensions (terminal uses cols/rows mapped to pixels by theme)
    - Add width/height from rendered frame dimensions (set at construction)
    - CursorState: {x, y, visible} (no scale — terminal cursor is fixed size)
    - HudState: {labels, opacity} (opacity for fade animation support)
    - Store fps, width, height, theme config (cursor + HUD styling), frames[], events[]
    - Use writeFileSync/readFileSync for save/load (same as webreel)
    - Re-export InteractionTimeline and types from packages/core/src/index.ts
  </implementation>
</feature>
